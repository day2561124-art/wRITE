# 長線骨架｜缺檔保護卡

本檔為專案長線骨架、伏筆邊界與未支付方向的啟用位置。

目前尚未匯入正式長線骨架母檔。依 `data/canon_db/active_engine.md` 與 `data/writing_policy_db/active_writing_card.md` 的現行分工，完整未支付長線骨架屬於外置專案指令，不在 Canon DB 或 Writing Policy DB 內重存。

## 目前狀態

- 正式長線骨架：尚未匯入。
- 本檔性質：缺檔保護卡，不是正式長線內容。
- 權限：只能防越界，不能新增正史、不能定案篇章方向、不能替代使用者專案指令。

## 缺檔時的硬邊界

在正式長線骨架匯入前，AI 不得自行推定或提前支付下列事項：

- 代表隊最終名單。
- 正式代表資格、正式名額、正式編組。
- 正式選拔第一日其他場次結果。
- 篇章級重大入選、失位、退賽或替補。
- 長期傷勢、長期失能、不可逆創傷或死亡。
- 重大能力突破、階段提升、固定勝法或穩定高階能力。
- 固定配對、重大關係翻轉、完整和解或戀愛成果。
- 暗線核心真相、組織核心目的、敵方介入結論。
- 重大危機終局、世界格局改寫或篇章級答案。
- 未支付技能方向被寫成已解鎖、已掌握、已公開或已取得成果。

## 起稿時的允許範圍

在缺少正式長線骨架時，候選正文只能做低風險承接：

- 承接 Canon DB 已成立之時間、場次、傷勢、短期限制與公開結果。
- 支付已成立事件的短期生活、醫療、訓練、排程、情緒或關係後座。
- 讓角色因已成立後果產生選擇成本，但不得把選擇推成長線定案。
- 使用 Writing Policy DB 決定場景怎麼寫，但不得讓寫作卡反向決定長線內容。
- 若任務需要篇章方向、危機濃度、角色線、競技線或禁止偏移，必須要求使用者提供正式長線骨架或本輪專案指令。

## 正式匯入方式

取得正式長線骨架母檔後，先 dry-run：

```powershell
node server/src/tools/import-policy-file.mjs --kind longline --source "E:\設定集\完整設定檔\長線骨架_v1.0.md" --dry-run
```

確認版本、SHA-256 與目標路徑無誤後，移除 `--dry-run` 正式匯入。

若來源檔名或內文沒有 `vX.Y` 版本號，必須手動指定：

```powershell
node server/src/tools/import-policy-file.mjs --kind longline --source "E:\設定集\完整設定檔\長線骨架.md" --version v1.0
```

## 對 AI 的操作規則

任務若碰到長線缺口，先輸出：

```text
目前尚未匯入正式長線骨架；以下事項不可推定為正史或長線定案。
```

然後列出本次任務涉及的不可越界項。不得用「合理推測」「敘事需要」「角色應該會」替代正式長線資料。
